# Rebrand: "Sadhguru Coaching Centre" → "Sadguru Coaching Classes"

The codebase currently uses "Sadhguru Coaching Centre" everywhere (already rebranded from "Mahima Academy" previously). The new target is **"Sadguru Coaching Classes"** (note: "Sadguru" not "Sadhguru", and "Classes" not "Centre").

## Scope of Changes

### 1. Global Text Replacement (~28 files)

Replace all instances of:

- `Sadhguru Coaching Centre` → `Sadguru Coaching Classes`
- `Sadhguru Coaching` → `Sadguru Coaching` (short form in mobile nav)
- `Sadhguru` → `Sadguru` (short_name in manifest, storage keys, CSS class names)
- `Mahima Academy` → `Sadguru Coaching Classes` (any remaining references in comments/docs)

**Key files affected:**

- `index.html` — title, meta tags, OG tags, apple-mobile-web-app-title
- `public/manifest.json` — name, short_name, description
- `src/pages/Index.tsx` — hero data, nav text
- `src/pages/Login.tsx`, `src/pages/Signup.tsx` — brand text
- `src/components/Layout/Header.tsx` — header brand name
- `src/components/Layout/Sidebar.tsx` — sidebar brand
- `src/components/Landing/Footer.tsx` — footer brand
- `src/pages/AdminUpload.tsx` — watermark default text
- `capacitor.config.ts` — appName
- All video player components: `WhiteLabelVideoPlayer.tsx`, `MahimaGhostPlayer.tsx`, `SecureVideoPlayer.tsx`, `PremiumVideoPlayer.tsx`, `PdfViewer.tsx`, `MahimaPlayer.tsx`, `MahimaVideoPlayer.tsx`

### 2. CSS Class & Storage Key Renames

- `.sadhguru-player` → `.sadguru-player` in CSS and component classes
- `sadhguru_player_volume` → `sadguru_player_volume` localStorage key
- `sadhguru_selected_batch` → `sadguru_selected_batch` localStorage key
- `.sadhguru-watermark` → `.sadguru-watermark` CSS class
- `.mahima-loader-logo` / `.mahima-loader-ring` → `.sadguru-loader-logo` / `.sadguru-loader-ring`

### 3. File/Component Names (rename references only, keep files)

The component files like `MahimaGhostPlayer.tsx`, `MahimaPlayer.tsx`, `MahimaVideoPlayer.tsx` will keep their filenames but internal comments will be updated. Renaming files would break too many imports across the codebase for minimal benefit.

### 4. Logo Assets

The existing logo files (`mahima-logo.png`, `mahima-academy-logo.png`) will continue to be used since no new logo files have been provided. A comment will be added where logo images are referenced to indicate they should be replaced when new assets are available.

### 5. PWA Manifest & Capacitor

- `public/manifest.json`: name → "Sadguru Coaching Classes", short_name → "Sadguru"
- `capacitor.config.ts`: appName → "Sadguru Coaching Classes"

### 6. Documentation

- Update `memorywork.md` with rebranding changelog entry
